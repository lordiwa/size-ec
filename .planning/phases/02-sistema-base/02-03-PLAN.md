<objective>
Close the three remaining behavioural gaps in the style/gate/footer trio:

1. **Post-gate routing** — currently the gate renders inline at App level and never restores the originally requested route. After the user picks a size, they should land on the URL they originally clicked (e.g. `/servicios`), not stay on the gate or fall through to whatever route was already in the address bar.

2. **M-tick reset semantics** — the M tick in the footer mini-slider is the canonical reset (per CONTEXT.md D-02). It already calls `setSize('M')`, which (via the existing watchEffect in App.vue) clears `has-market` and `market-{id}` because the watcher rebuilds class names from the new active state. This task confirms the semantics, adds the visual accent ring when a non-M style is active, and ensures the no-op-on-fresh-visit behaviour holds (today, clicking M on a fresh visit writes `{type:'size', value:'M'}` to localStorage — that's a regression because it would make the gate disappear; we need to skip the call when the visitor has no prior flag and is already on M-default).

3. **Gate visibility on protected routes** — currently the gate replaces `<RouterView>` only when `gateRequired` is true. We keep that pattern but add the intended-route capture so the gate "remembers" where the user wanted to go.

Purpose: Honors REQ-style-gate (gate appears on protected routes for first-time visitors and returns them to the requested view after pick) and REQ-style-persistence (single flag, last write wins). Honors CONTEXT.md D-02 (M tick IS the reset).

Output: `src/App.vue` captures `route.fullPath` when the gate triggers and calls `router.replace(intendedPath)` once `style.active` becomes non-null. `src/components/StickyFooter.vue` shows the M-tick accent ring when a non-M style is active and skips the `setSize('M')` write when the visitor is already on M-default with no flag. `src/components/StyleGate.vue` is unchanged in behaviour — it still calls `style.setSize(size.id)`, the App.vue logic handles the rest.

```typescript
import { computed } from 'vue'

const style = useStyleStore()

const navItems = [
  { to: { name: 'quienes-somos' }, label: 'Quiénes somos' },
  { to: { name: 'servicios' }, label: 'Servicios' },
  { to: { name: 'contacto' }, label: 'Contacto' }
]

const sizes: SizeId[] = ['XS', 'S', 'M', 'L', 'XL']

function isActiveSize(s: SizeId): boolean {
  if (style.active?.type === 'size') return style.active.value === s
  return s === 'M' && !style.active
}

// True when a non-M style (any market, or any size != M) is active.
// In that state, the M tick acts as the reset control and gets a faint accent ring.
const resetMode = computed<boolean>(() => {
  if (!style.active) return false
  if (style.active.type === 'market') return true
  return style.active.value !== 'M'
})

function pickSize(s: SizeId): void {
  // No-op guard: if the visitor is on M-default with no flag and clicks M,
  // do not write a flag — that would silently bypass the gate on protected routes.
  if (s === 'M' && !style.active) return
  style.setSize(s)
}
```
