  <action>
    Create `scripts/check-contrast.cjs`. The script:

    1. Hard-codes the M token table (BG=#0a0a0a, INK=#f5f5f5, MUTED=#8a8a85, LINE=#ffffff14, LINE_STRONG=#ffffff33, ACCENT=#ff3b1f). These are derived from the prototype and present in `src/styles/main.css` lines 60+. Document the source comment-line so Phase 3 can extend.
    2. Implements WCAG 2.1 relative luminance + contrast ratio per the spec (sRGB → linear → L = (R*0.2126 + G*0.7152 + B*0.0722); contrast = (L1+0.05)/(L2+0.05) where L1 ≥ L2). Handles 8-digit hex (drops alpha for the contrast math; emits a warning if alpha < 1.0 because the visual contrast will depend on the underlying layer).
    3. Computes and prints a table of pairs:
       - body (INK on BG) — must be ≥ 4.5
       - muted (MUTED on BG) — must be ≥ 4.5 because muted text is still copy
       - large heading (INK on BG) — must be ≥ 3.0 (always passes if body passes)
       - accent CTA (BG on ACCENT) — must be ≥ 4.5 (the .bright-cta uses bg color on accent background)
       - accent inline text (ACCENT on BG) — must be ≥ 3.0 (used for accent words inside body copy at large sizes)
    4. Exits 0 if all required pairs meet their threshold; non-zero with a clear message naming the failing pair otherwise.
    5. Prints a concise summary: `M Crafted dark-mode | 5 pairs checked | all ≥ 4.5 (lowest: muted/bg = X.XX:1)` so the contract is visible.

    Add `"check:contrast": "node scripts/check-contrast.cjs"` to `package.json` "scripts" block. Place it next to `type-check` for discoverability.

    Do NOT introduce a runtime dependency. Pure Node, no npm installs.
  </action>
