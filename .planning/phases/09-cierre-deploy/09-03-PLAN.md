    ```markdown
    ---
    name: deploy
    description: Commit, push, type-check, build, and deploy the SIZE site to Firebase Hosting. Takes a commit message as input. Aborts before publish if type-check or build fails. Use this whenever the user wants to ship the current working tree.
    ---

    # Deploy skill — SIZE

    You are running the canonical push-to-deploy chain for the SIZE Vue 3 + Vite site.

    ## Argument

    The user provides a commit message as a free-form string. If they did not provide one, stop and ask for it before doing anything. Never invent a commit message.

    ## Pre-flight

    1. Confirm working directory is the repo root (must contain `package.json` AND `firebase.json` AND `.firebaserc`). If not, abort with a clear message.
    2. Determine the current branch with `git rev-parse --abbrev-ref HEAD`. Capture the value as `BRANCH`.

    ## Chain (run in order; abort at the first failure)

    Run each step. If any step exits non-zero, STOP — do not continue down the chain — and surface the failure to the user with the failing step name and the relevant tail of stderr.

    1. **Stage** — `git add -A`
    2. **Commit** — `git commit -m "<the user-provided commit message>"`. If `git commit` reports `nothing to commit, working tree clean`, treat that as a soft-success and continue (the user may want to redeploy an already-committed state); print a one-line note.
    3. **Push** — `git push origin <BRANCH>`. If the branch has no upstream, run `git push -u origin <BRANCH>` instead. Network/auth failures abort the chain.
    4. **Type-check (gate)** — `pnpm type-check`. Must exit 0. On failure, abort BEFORE build and BEFORE deploy. Do not amend the commit; the failure is reported and the user fixes it locally.
    5. **Build (gate)** — `pnpm build`. Must exit 0 and produce `dist/index.html`. On failure, abort BEFORE deploy.
    6. **Deploy** — `pnpm exec firebase deploy --only hosting`. The Firebase project is read from `.firebaserc` (`projects.default`). If this is the first deploy on this machine, firebase-tools may prompt for `firebase login` — surface that to the user and pause (the user authenticates in their browser, then re-invokes the skill). Do NOT attempt to script around the login flow.

    ## Post-flight

    - Capture the hosting URL printed by `firebase deploy` (it ends with `.web.app`).
    - Print a 3-line summary to the user: branch pushed, commit SHA, hosting URL.

    ## Hard rules

    - Order is non-negotiable: stage → commit → push → type-check → build → deploy.
    - Type-check and build are gates. Both must pass before `firebase deploy` is invoked.
    - Never run `firebase deploy` without a green build immediately preceding it in the same skill invocation.
    - Never commit secrets. The Atlassian API token lives in the shell environment, not in the repo (see CLAUDE.md → Atlassian API token setup).
    - Never auto-bump versions or change `package.json` as part of this skill.
    - This skill is the canonical deploy. Do not propose alternative ad-hoc deploy paths during execution.
    ```
