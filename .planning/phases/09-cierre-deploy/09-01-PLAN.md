  </read_first>
  <action>
    Create `firebase.json` at repo root with this exact content:

    ```json
    {
      "hosting": {
        "public": "dist",
        "ignore": [
          "firebase.json",
          "**/.*",
          "**/node_modules/**"
        ],
        "rewrites": [
          {
            "source": "**",
            "destination": "/index.html"
          }
        ],
        "headers": [
          {
            "source": "**/*.@(js|css|woff|woff2|ttf|otf)",
            "headers": [
              { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
            ]
          },
          {
            "source": "**/*.@(html)",
            "headers": [
              { "key": "Cache-Control", "value": "no-cache" }
            ]
          }
        ]
      }
    }
    ```

    The SPA catch-all rewrite (`"source": "**"` → `/index.html`) is non-negotiable per CONTEXT.md — Vue Router deep-link refresh depends on it.

    Create `.firebaserc` at repo root:

    ```json
    {
      "projects": {
        "default": "size-ec"
      }
    }
    ```

    Use project id `size-ec`. If the Firebase project does not yet exist under that id, the human creates it in Plan 04 (Firebase Console) using exactly this id; until then, this binding is a forward declaration — Plan 04 will reconcile.

    Append `.firebase/` to `.gitignore` if it isn't already there (firebase-tools writes a runtime cache to `.firebase/` that should not be committed).
  </action>
  <verify>
    <automated>node -e "const j=require('./firebase.json'); if(j.hosting.public!=='dist')process.exit(1); if(!j.hosting.rewrites.some(r=>r.source==='**'&&r.destination==='/index.html'))process.exit(2); const r=require('./.firebaserc'); if(!r.projects||!r.projects.default)process.exit(3); console.log('ok');"</automated>
